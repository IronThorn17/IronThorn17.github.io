Scope 360 IFC Asset Detection

A machine learning pipeline and UI system for automatically detecting and labeling assets in IFC construction files, developed for Immersion Data Solutions.

Screenshots

Your Role

Developed major components of the UI and machine learning pipeline. Set up the project architecture using Docker, React, Node.js, and a PostGIS-enabled database. Worked closely with teammates to integrate YOLO-based object detection into the workflow.

Tools & Technologies

  • React
  • Node.js
  • Docker
  • YOLO Machine Learning Model
  • Python
  • PostgreSQL + PostGIS

Type of Project

Industry-sponsored senior project (team-based)

Overview

This project aimed to automate the identification of construction assets inside IFC models by applying machine learning to classify and label objects. Our team was tasked with building a full pipeline: frontend interface, backend processing logic, a database for storing labeled assets, and a YOLO-based detection model wrapped in a Dockerized environment to ensure portability across systems.

Challenges & Problems Solved

  • Designed the entire project architecture using tools that were unfamiliar at the start, including Docker, PostGIS, and YOLO—requiring rapid self-learning and technical adaptation.
  • Implemented the frontend UI in React to upload, view, and manage parsed IFC data, enabling engineers to easily interact with model information.
  • Integrated a YOLO machine learning workflow using Python, including model loading, inference execution, and data formatting for the backend.
  • Created a Docker-based environment to unify the frontend, backend, ML pipeline, and database into a consistent deployment system.
